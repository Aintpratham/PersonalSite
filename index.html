03 / FLAGSHIP PROJECT

Nova — a job-application automation platform.

My biggest build: a Chrome extension paired with a cloud backend that detects and applies to warehouse shifts on Amazon's hiring platform in near real-time. Sold to real, paying users — and kept alive through real production failures.

● LIVE PRODUCT · MONETIZED
Nova
Founder & Lead Developer · 2024 – Present · C++, Python, JavaScript / Node.js

A hybrid Manifest V3 Chrome extension + Node.js backend that automates multi-step application workflows on Amazon's hiring platform. I iterated across 13+ versions over ~6 months, owning everything from protocol-level integration and resilient polling to licensing, deployment, pricing, onboarding, and support.

// what it does under the hood
  • ▸Integrated with the platform's GraphQL API instead of brittle UI scraping — then drove the site's own single-page app to fire the create-application request, with an idempotent direct-POST fallback guarded by single-flight de-duplication to prevent double submissions.
  • ▸Built a resilient polling engine: parallel ~1s targeted polls with jitter, exponential backoff (capped) on consecutive failures, and explicit handling of auth (401/403) and rate-limit (419/429) responses — staying within the platform's WAF limits while detecting new listings in roughly one round-trip.
  • ▸Token / session authentication reading rotating access tokens from cookies with short-lived caching and automatic invalidation on 401, plus connection pooling and a concurrency-limited dispatcher with retries and time-bounded dedup.
  • ▸A full license server (Node.js / Express on Railway + Firestore) with device fingerprinting, an admin web console, soft install caps with automatic aging, and per-device block / pause / revoke controls.
// how it evolved
PHASE 1 — NATIVE
C++ desktop licensing layer
Locked the software to each machine via a hardware ID — the motherboard serial read through WMI, hashed with the license key using the Windows CryptoAPI.
PHASE 2–3 — AUTOMATION
Python + Selenium engine
Coordinator-driven browser automation that drove the apply flow and handled the multi-stage application states.
PHASE 4+ — CLOUD
Node.js cloud backend
Migrated licensing & logic to a centralized server (Railway + Firestore) for scalable, remote control over distribution.
v11–v13 — REAL-TIME
MV3 extension + EC2 + GraphQL
Browser-side apply logic with server-side targeted polling over an authenticated WebSocket — the current architecture.
